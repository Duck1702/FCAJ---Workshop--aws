# Event 1: Cloud Architect

## Event Information

**Event name:** Cloud Architect

**Time:** June 20, 2026

**Location:** 26th Floor, Bitexco Tower, 02 Hai Trieu Street, Sai Gon Ward, Ho Chi Minh City

**Role in the event:** Participant

## Event Objectives

Cloud Architect was organized as a game show to create a more engaging learning environment and help participants strengthen their knowledge of Cloud, AWS, and system architecture design.

The event did not only focus on answering theoretical questions. It also helped participants practice problem analysis, quick thinking, teamwork, and applying cloud knowledge to practical situations.

## Event Description

In the Cloud Architect event, teams competed against each other through multiple rounds. Each team answered questions from different question sets, ranging from easy to difficult levels. The team with the higher score advanced to the next round.

If both teams had the same score at the end of a round, an additional question was given. The team that answered faster would win. The questions were mainly related to Cloud, AWS, and architecture design knowledge.

The event had a limited number of participants, with 8 teams in total and 5 members per team. Team members did not necessarily need to be from the same internship group, which created an opportunity to connect and collaborate with other participants in the program.

## Main Activities

* Reviewed basic Cloud knowledge.
* Answered questions related to AWS and cloud architecture.
* Analyzed architecture design situations using a cloud-based approach.
* Worked as a team to discuss and agree on answers within a limited time.
* Practiced quick thinking when handling technical questions.
* Learned more about important architecture factors such as security, scalability, availability, and cost control.

## Special Skills in the Game Show

During the competition, each team could use special skills to increase the strategic aspect of the game.

**Minimum Risk:**
The team could use this skill when they were not confident about an answer. If the answer was wrong, no points would be deducted. If the answer was correct, the team would receive only half of the question’s points.

**Hope Star:**
The team could use this skill for the question they were most confident about. If the answer was correct, the score would be doubled. However, if the answer was wrong, the penalty would also be doubled.

These skills made the event more exciting and required teams to evaluate their confidence level, manage risks, and make reasonable decisions.

## Results Achieved

After joining the event, I had the opportunity to review and reinforce important knowledge about Cloud and AWS. Through the questions in the game show, I realized that designing a cloud system is not only about choosing the right AWS services, but also about considering security, cost, scalability, and long-term operations.

The event also helped me improve teamwork, discussion, and decision-making skills under time pressure. These are important skills when working on cloud architecture-related projects.

## Lessons Learned

* It is important to understand fundamental Cloud concepts before designing a system on AWS.
* When choosing AWS services, several factors need to be considered, such as cost, security, performance, and scalability.
* Teamwork helps make the analysis and answering process more effective.
* In some situations, risk management is necessary instead of choosing answers based only on intuition.
* Cloud architecture knowledge is very important when implementing real-world systems.

## Applying the Knowledge to the Project

The knowledge gained from the Cloud Architect event can be directly applied to my project, **AI-Powered Personal Finance Budget Alert System on AWS**. When designing the system, I need to pay attention to selecting a suitable architecture, prioritizing serverless services to reduce operational costs and improve scalability.

In addition, the topics related to cost control, security, and architecture design support my plan to implement services such as Amazon Cognito, API Gateway, AWS Lambda, DynamoDB, SNS, CloudWatch, Bedrock, and AWS Budgets.

## Personal Experience

This was an interesting event because the competition format made learning Cloud more engaging. Instead of only studying theory, I was able to participate in situations that required quick thinking, team discussion, and answering questions within a limited time.

Through this event, I gained more motivation to continue learning AWS and cloud architecture, especially how to design systems that are efficient, secure, scalable, and cost-effective.
