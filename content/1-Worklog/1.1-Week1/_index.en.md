## Difficulties encountered

* AWS has many services and new concepts, so it took time to become familiar with them.
* It was necessary to clearly distinguish when to use the root user and when to use an IAM user.
* AWS costs must be monitored carefully because some services may generate charges if they are configured incorrectly or not deleted after use.
* Kiro has a free credits limit, so it needs to be used efficiently.

## Solutions

* Took notes on important concepts during the learning process.
* Used the root user only for initial administrative tasks and planned to use an IAM user for daily work.
* Learned about services that may generate costs before creating AWS resources.
* Avoided using Kiro for parts that could be written manually, and planned to use it mainly for coding support and debugging.
